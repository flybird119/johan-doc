Real-time monitoring

Monitor the statuses and real-time events of doors under the access control panels in the system in real-time, including normal events and exceptional events (including alarm events).

Appendix: Real-Time Event Description:

Normal punching card open: Means the person has open door permission punch the card and trigger this normal event of open the door .
Punching card at door normally open period: At the normally open period (set to normally open period of a single door or the door open period after the first card normally open), the person has open door permission punch the effective card at the opened door to trigger this normal events.
Punching card as first card normally open: The person has first card normally open permission, punch card at the setting first card normally open period but the door is not opened, and trigger the normal event.
Multi-card open: Multi-card combination can be used to open the door. After the last piece of card verified, the system trigger this normal event.
Emergency password open door: The password (also known as the super password) set for the current door can be used for door open. It will trigger this normal event after the emergency password verified.
Open door at normally open period: If the current door is set a normally open period, the door will open automatically after the setting start time, and trigger this normal event.
Trigger linkage events: After the system linkage configuration take effect, trigger this normal event.
Cancel alarm: When the user cancel the alarm of the corresponding door, and the operation is success, trigger this normal event.
Remote open door: When the user open a door from remote and the operation is successful, it will trigger this normal event.
Remote close door: When the user close a door from remote and the operation is successful, it will trigger this normal event.
Cancel normally open: Punch effective card 5 times or remote close the door to cancel the current state of the door normally open, trigger this normal event.
Card punching interval is too short: when the interval between two card punching is less than the set time interval, trigger this abnormal event.
Card punching out of door effective period: The user has the door open permission, punch card but not at the door effective period of time, and trigger this abnormal event.
Illegal time period: The user has the door open permission, punch card but not at the access effective period of time, and trigger this abnormal event.
Unauthorized access: The registered card without the access permission of the current door, punch to open the door, trigger this abnormal event.
Anti-pass back: When the anti-pass back setting of the system take effect, trigger this abnormal event.
Interlock: When the interlocking rules of the system take effect, trigger this abnormal event.
Multi-card verify: Use multi-card combination to open the door, the card verification before the last card (whether verified or not), trigger this normal event.
Card is not registered: Refers to the current card is not registered in the system, trigger this abnormal event.
Door open expired: The door sensor detect that it is expired the delay time after opened, if not close the door, trigger this abnormal event.
Card has expired validity: The person with the door access permission, punch card to open the door after the effective time of the access control, can not be verified and will trigger this abnormal event.
Duress password open door: Use the duress password of current door verified and triggered alarm event.
Door is unexpectedly opened: Except all the normal events (normal events such as user with door open permission to punch card and open the door, password open door, open the door at normally open period, remote door open, the linkage triggered door open), the door sensor detect the door is opened, that is the door is unexpectedly opened.
Door is opened: When the door sensor detect that the door has been properly opened, triggering this normal event.
Door is closed: When the door sensor detect that the door has been properly closed, triggering this normal event.
Exit button open: When using the door button to open the door, triggering this normal event.
Auxiliary input point disconnect: When the auxiliary input point disconnected, trigger this normal event.
Auxiliary input point short: When the auxiliary input point short circuit, trigger this normal event.
