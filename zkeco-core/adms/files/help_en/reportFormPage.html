Access control reports


Includes [All access control events], [Access control exception events] and [Personnel access levels] reports. You can select Export all and Export after query.

The user can generate statistics of relevant device data as per reports, including card verification information, door operation information, and normal card punching information, etc.

Normal event: Such as Normal Punching Card Open, Punching Card at Door Normally Open period, Punching Card as First Card Normally Open, Multi-Card Open, Emergency Password Open Door, Open Door at Normally Open Period, Trigger Linkage Events, Door is Opened, Door is Closed, Exit Button Open, Auxiliary Input Point Disconnected, Auxiliary Input Point Short, etc.;

Exceptional Event: Such as Card Punching Interval is too Short, Card Punching Out of Door Effective Period, Illegal Time Period, Unauthorized Access, Anti-Passback, Interlock, Multi-Card Verify, Card is not registered, Door Opening Expired, Card has Expired Validity, Duress Password Open Door, Door is Unexpectedly Opened, Duress Password Open Door, Door is Unexpectedly Opened are alarm events.

Verify mode: Only Card, only password, password plus card and other s.

Note: Only event records generated when the user uses emergency password to open doors will include [Only password] verification mode.
